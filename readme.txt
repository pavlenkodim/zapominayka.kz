Проектная работа
Приложение: Напоминалка о правилах по предметам
Механика приложения: В течении дня на устройство должно приходить уведомление о повторении правила, даты/события или теоремы.Для начала создать на один класс 
Перметы для повторения должен выбрать пользователь на свой вкус, интерес или необходимость, чтобы подтянуть знания по предмету.
Этапы разработки WEB приложения:
1) Сбор информации и всех правил по пердмету один в документ;
2) Создание базы данных;
3) Создание и программирование Desctop приложения в IDLE;
4) Создание графического пользовательского инетерфейса GIU:
    Прорисовка фонов, выбор шрифтов, прорисовка спрайтов, кнопок, значков, иконок, выбор стилистики;
5) Внедрение GUI в проект;
6) Изучение и внедрение библиотек для дальнейшей разработки и тестирования;
7) Окончательная доработка и тестирование на Dectop;
8) Портирование приложения на Android и iOS;
9) Тестирование приложения на различных моделях устройств: телефонах, планшетах, ПК;
10) Создание документации для проекта: Цели, гипотезы, практическая занчимость, план работы и т.д;
11) Подготовка к защите проектной работы: презентация;
12) Защита работы на конкурсе проектных работ
    Победа в конкурсе (забрать Гранд-при) + ?призовые? + репутация + перспективы в дальнейшем трудоустройстве

В конце недели можно предлагать тест по пройденному материалу, а так же записывать прогресс каждого пользователя и предлагать что нудно повторить и где он ошибся.
